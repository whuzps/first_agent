# 部署说明文档

## 目标与范围
- 说明如何在开发/测试/生产环境部署 `first_agent` 服务，完成依赖安装、配置、初始化、启动与验证。
- 适用组件：FastAPI 应用、LangGraph 对话引擎、Milvus 向量库、Redis 会话缓存、PGSQL 订单库、SSE 建议推送、日志与指标。

## 环境要求
- 操作系统：Windows 10/11、Linux（x86_64）或 macOS（ARM/Intel），
- 运行时：Python ≥ 3.11（推荐 3.11/3.12）
- 依赖工具：`pip`、`virtualenv`、`git`；可选 `Docker`/`docker compose`
- 端口占用：
  - `8000`（FastAPI 应用）
  - `6379`（Redis Stack，用于语义缓存）
  - `9090`（Prometheus metrics）
  - `19530`（Milvus 向量库）
  - `9001`（MinIO 控制台，Milvus 存储后端）

## 依赖安装

### 方式一：本地 Python 环境
- 创建并激活虚拟环境：
  - Windows: `python -m venv .venv && .\.venv\Scripts\activate`
  - Linux/macOS: `python -m venv .venv && source .venv/bin/activate`
- 安装依赖：
  - `cd service && pip3 install -r requirements.txt`

### 方式二：Docker Compose（推荐一体化）
- 启动全部服务（应用 + Redis Stack + Milvus）：
  ```bash
  cd service
  docker compose up -d
  ```
- 仅启动基础设施（Redis Stack + Milvus），应用本地运行：
  ```bash
  docker compose up -d redis etcd minio milvus
  ```
- 查看日志：`docker compose logs -f app`
- 停止：`docker compose down`

## 配置项
- 在 `service/` 目录下创建 `.env` 文件（或设置系统环境变量），关键变量：

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `MODEL_NAME` | `qwen-plus` | 对话模型名称 |
| `EMBEDDING_MODEL` | `text-embedding-v4` | 嵌入模型名称 |
| `DASHSCOPE_API_KEY` | （无） | 通义千问 API Key（必填） |
| `MILVUS_HOST` | `localhost` / Docker 中 `milvus` | Milvus 地址 |
| `MILVUS_PORT` | `19530` | Milvus 端口 |
| `COLLECTION_NAME` | `rag_test_v1` | RAG 向量集合名 |
| `REDIS_URL` | `redis://127.0.0.1:6379/0` | Redis 连接地址 |
| `JWT_SECRET` | `cs-agent-jwt-secret-2025` | JWT 签名密钥（生产环境务必修改） |
| `TENANTS_BASE_DIR` | `service/tenants` | 多租户数据根目录 |
| `ORDERS_DB_PATH` | 自动按租户定位 | 订单数据库路径 |
| `CHECKPOINT_DB_PATH` | 自动按租户定位 | LangGraph 检查点数据库 |
| `SUPPORT_DB_PATH` | `service/db/support.db` | 未命中问题记录库 |
| `HUMAN_SUPPORT_URL` | （无） | 人工兜底渠道 URL |
| `LANGCHAIN_TRACING_V2` | `false` | 是否启用 LangSmith 追踪 |
| `LANGCHAIN_API_KEY` | （无） | LangSmith API Key |

## 初始化步骤
- 订单库初始化（测试数据）：
  - 本地：`python -m core.init_orders_db`（在 service/ 目录下执行）
  - Docker：`docker compose exec app python -m core.init_orders_db`
- 日志推送守护（可选）：
  - `python core/log_push.py --daemon --config logs/log_push_config.json`
  - 验证：查看 `logs/log_push_service.log`

## 启动服务
- 本地启动：
  `uvicorn main:app --reload --host 0.0.0.0 --port 8000`
  `python3 service/app/main.py`
- celery进程启动: `celery -A celery_app worker --queues=chat_queue --concurrency=4 --loglevel=info`
- Docker 启动：`cd service && docker compose up -d`

## 验证方法
- 健康检查：`curl http://localhost:8000/health`
- Milvus 集合列表：`curl http://localhost:8000/api/v1/milvus/collections`
- Prometheus 指标：`curl http://localhost:8000/metrics`

## 故障排查
- 应用不可用：检查 `docker compose logs app` 或本地控制台；验证 `/health`。
- Milvus 连接失败：确认 Milvus 容器健康 `docker compose ps`；检查 `MILVUS_HOST` / `MILVUS_PORT`。
- SSE 超时：前端需保持连接，服务端默认 15s 超时；查看 `SUGGEST_QUEUES` 是否有事件。
- 日志推送失败：查看 `log_push_service.log` 与 `log_push_config.json` 鉴权参数。

## 日志与监控
- 应用日志：`logs/requests.log`（包含耗时、模型切换、向量增删审计）
- 指标快照：`GET /health` 返回 `overall/kb/order/direct/handoff/vectors_*` 的最值与分位统计
- Prometheus 拉取：`GET /metrics`（集成到 FastAPI，也可通过独立 9090 端口拉取）
- 外部推送：`log_push.py` 守护将日志以 NDJSON 上报 ELK/Logstash
